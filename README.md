# vibe-check-dc604

Source for the hands-on labs from **Vibe Check: Exploiting Trust — from
Developers to AI agents** (DC604, June 2026). Published so you can read
through them at your own pace.

> ⚠️ **Read these in a sandbox — don't run them on your real machine.**
> Treat any unfamiliar repo as hostile until you've looked it over; that
> habit *is* the lesson. These copies are defanged for the workshop, but
> read them like the real thing.

## Labs

- **`lab1/`** — the two take-home "assessment" repos a candidate is handed in
  the talk's fake-recruiter scenario: a Node/Express auth service
  (`assessment-1/`) and a small Node utility (`assessment-2/`), each with a
  plausible bug to fix.
- **`lab2/`** — a minimal AI coding-agent harness (`harness/`) and the set of
  repos it reviews (`repos/`), used to walk through where an autonomous agent
  places its trust.

Each folder reads like ordinary dev work. Whether that's all it is, is left
for you to find out — the full walkthrough is in the talk.
